`/debug`는 성능 측정 및 함수의 경로를 추적하는 용도로 사용됩니다.

# 명령어
#### `/debug start`
성능 측정을 시작합니다.

#### `/debug stop`
성능 측정을 종료하고 시간을 표시합니다.

#### `/debug function <name>`
함수를 실행하여 함수 내에 있는 명령어에 대한 정보를 얻습니다.  
`/return run` 및 다른 함수에서 실행할 수 없습니다.

결과는 `.minecraft/debug` 폴더에 결과가 출력되며, 파일의 이름은 `debug-trace-yyyy-MM-dd_HH.mm.ss.txt`입니다.

<br/>

# `.txt` 구조
| 기호 | 설명 |
|------|------|
| [C]        | 실행된 명령어 |
| [M]        | 명령어 실행에 대한 메시지 |
| [E]        | 실패한 명령어에 대한 메시지 |
| [F]        | 함수에 존재하는 모든 명령어 줄의 합 |
| [R = Int]  | 함수에서 실행된 명령어의 수 |
| [C -> Int] | 명령어가 실행되고 명령어의 반환 값 |

### `example.txt`
```js
<함수 이름>
   [F] <함수 이름> size=<모든 명령의 줄 수>
      [C] title @s actionbar "test" // 실행된 명령어
         [M] QUAM12에게 새로운 액션바 제목을 보여줍니다 // 명령어 실행에 대한 메시지
      [R = 1] title @s actionbar "test" // 플레이어 1명이라 R=1
      [C] scoreboard players set #test test 1
         [M] #test의 [test]을(를) 1(으)로 설정했습니다
      [R = 1] scoreboard players set #test test 1
      [C] execute if entity @s[tag=!test]
         [M] 테스트가 통과했습니다, 개수: 1
      [R = 1] execute if entity @s[tag=!test]
      [C] execute if entity @s[tag=test]
         [E] 테스트가 실패했습니다                [M] 테스트가 실패했습니다 // 실패한 명령어에 대한 메시지
      [C] execute if entity @s[tag=test]
         [E] 테스트가 실패했습니다                [M] 테스트가 실패했습니다
      [C] execute if entity @e[type=armor_stand]
         [M] 테스트가 통과했습니다, 개수: 5 // 명령어 실행에 대한 메시지
      [R = 5] execute if entity @e[type=armor_stand] // 세계에 갑옷 거치대가 5개 있으므로 R=5
      [C] say test -> 1 // say가 실행되고 반환 값
      [C] say test -> 1
      [C] say test -> 1
      [C] say test -> 1
      [C] say test -> 1
      [C] return 1
```
