<h1 style="font-family: Minecraft-Event"/>Data Pack Version 91.0</h1>

- World Border **interpolation** is now dependent on game **ticks** instead of **real time**  
   <span style="font-size:16px; color: #cccccc">(월드 보더의 **보간**은 이제 **현실 시간**이 아닌 게임 **틱**에 귀속됩니다)</span>
   - This means the World Border will no longer move when the game is paused  
      <span style="font-size:16px; color: #cccccc">(게임이 일시 정지되면 월드 보더가 더 이상 움직이지 않습니다)</span>
   - It will also respect any changes made by the `/tick` command  
      <span style="font-size:16px; color: #cccccc">(<code><span style="color: #ff9999">/tick</span></code> 명령어로 <code><span style="color: #ff9999">rate</span></code>를 변경하면 그에 맞춰 움직임이 조정됩니다)</span>
</details>

<br/>

월드보더 타이머는 시간차를 `ms` 단위로 측정할 수 있습니다.  
왜냐하면, 월드보더는 틱에 귀속되어 있지 않습니다. [[:mojang: MC-53491](https://bugs.mojang.com/browse/MC/issues/MC-53491)]

```java
@Override
public double getSize() {
   double d0 = (Util.getMillis() - this.lerpBegin) / this.lerpDuration;
   return d0 < 1.0 ? Mth.lerp(d0, this.from, this.to) : this.to;
}
```

월드보더 크기를 변경하는 중에 `/worldborder get` 명령어를 사용하면 밀리초 단위로 크기가 표시됩니다.

```mcfunction
worldborder set 59998968
worldborder set 59999968 1

# 특정 명령어 실행

worldborder get

worldborder set 59999968
```

주로 일정한 랙을 일으켜 다른 곳에 활용하는 용도로 사용됩니다.
