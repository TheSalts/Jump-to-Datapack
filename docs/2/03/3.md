여러 엔티티들 중에서 점수 값이 같은 엔티티나  
명령어 실행자 즉, 자기 자신을 예외하거나 자기 자신만을 포함할 때 사용할 수 있습니다.

스코어 보드를 사용하므로 필요한 점수판을 생성합니다.
```mcfunction
/scoreboard objectives add id dummy
```

먼저 엔티티들마다 서로 다른 점수를 부여해야 하는데, 이때 점수는 중복될 수 없습니다.

다음 명령어를 사용하면 서로 다른 점수를 부여할 수 있습니다.
```mcfunction
execute as @e[tag=entities] run function <id>

# id.mcfunction
   scoreboard players add #id id 1
   execute unless score @s a matches ..2147483647 run scoreboard players operation @s id = #id id
```

같은 점수를 가지고 있는 엔티티를 찾기 위해 우선 자신과 같은 점수의 표지를 소환합니다.
```mcfunction
/execute summon marker run scoreboard players operation @s id = @p id
```

## execute 중첩 사용
점수를 비교하는 구문인 `/execute if/unless score`의 경우, 한 개체만 제한된다는 단점이 있습니다.  

다음 명령어를 사용하면 같은 점수를 가진 엔티티를 찾을 수 있습니다.
```mcfunction
/execute as @e[type=marker] at @e[tag=entities] if score @s id = @n[tag=entities] id run say @s
```

예를 들어, 표지가 2개 entities 태그를 가진 엔티티가 2개 있다고 가정하면 흐름은 다음과 같습니다.

- marker1  
   - 엔티티1의 점수와 비교
   - 엔티티2의 점수와 비교

- marker2  
   - 엔티티1의 점수와 비교
   - 엔티티2의 점수와 비교

연산량은 `as`와 `at`을 곱하므로 엔티티의 수가 많아지면 성능 저하가 심해집니다.
