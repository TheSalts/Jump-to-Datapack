디버그 화면(`debug screen`)은 `F3` 단축키로 접근할 수 있으며, 플레이어의 좌표/시선과 블록, 청크, 엔티티, 메모리 점유율 등 게임 내 다양한 정보를 표시하는 일종의 창(`window`)입니다.

# 표시 정도
- `꺼짐`: 정보가 보이지 않음
- `오버레이에서`: 정보가 `F3` 상태에서만 표시됨
- `항상`: 정보가 항상 표시됨

# 옵션
## Biome
플레이어가 현재 있는 바이옴을 표시합니다.
```js
"minecraft:plains"
```

## 청크
- 섹션: `16x16x16` 크기의 정육면체로 한 개의 청크에 `24`개가 있습니다.
   - 해당 섹션에 블록이 하나라도 있어야 집계됩니다.
   - `-64 ~ 320` 총 `384` 블록, `384/16 = 24` 섹션

### chunk_render_stats
청크 렌더와 관련한 정보를 표시합니다.
```js
C: 10/600 (s) D: 2, pC: 000, pU: 00, aB: 06
```
- `C`: 보이는 섹션의 수/전체 섹션의 수 (s)
- `D`: 현재 렌더 거리
- `pC`: 섹션 로딩이 끝날 때까지의 남은 섹션 수
- `pU`: GPU 업로드 대기 상태의 청크 수
- `aB`: 알 수 없음

### chunk_source_stats
청크의 세부정보와 관련한 정보를 표시합니다.

## 엔티티
### entity_render_stats
렌더링 가능한 엔티티의 수를 표시합니다.
```js
E: 0/1, SD: 5
```
- `E`: 알 수 없음/현재 차원의 렌더링 된 모든 엔티티
- `SD`: 시뮬레이션 거리

## fps
현재 `fps`를 표시합니다.
```js
60 fps T: inf vsync
```
- `n fps`: 초당 프레임
- `T`: 최대 프레임률(`inf`: 무제한), 수직 동기화 여부

## game_version
현재 게임 버전을 표시합니다.
```js
Minecraft 1.21.11 (1.21.11/vanilla)
```

## gpu_utilization
그래픽 카드 점유율을 표시합니다.
```js
GPU: 12%
```

## heightmap
하이트 맵과 관련한 정보를 표시합니다.
```js
CH: S: 0  M: 0  ML: 0
SH: S: 0  O: 0  M: 0  ML: 0
```
- `S`: WORLD_SURFACE
- `O`: OCEAN_FLOOR
- `M`: MOTION_BLOCKING
- `ML`: MOTION_BLOCKING_NO_LEAVES

## light_levels
현재 플레이어의 좌표에서 밝기 레벨을 표시합니다.
```js
Client Light: 15 (15 sky, 0 blocks)
```

## local_difficulty
현재 차원의 난이도를 표시합니다.
```py
Local Difficulty: 0.00 // 0.00 (Day 0)
```

## looking_at_block
현재 바라보고 있는 블록의 정보를 표시합니다.
```js
Targeted Block: 0, 0, 0
minecraft:oak_sign
rotation: 0
waterlogged: false
#minecraft:signs
#minecraft:wall_post_override
#minecraft:all_signs
#minecraft:mineable/axe
#minecraft:standing_signs
```
- `Targeted Block`: 최대 거리 `20` 블록의 좌표
- `id`: 블록의 이름
- `property`: 블록의 상태
- `tags`: 블록이 속한 `tags` 그룹

## looking_at_entity
현재 바라보고 있는 엔티티의 정보를 표시합니다.
```js
Targeted Entity
minecraft:mannequin
```
- `Targeted Entity`: 최대 거리 `entity_interaction_range`
- `type`: 엔티티의 유형

## looking_at_fluid
```js
Targeted Fluid: 0, 0, 0
minecraft:empty
```
```js
Targeted Fluid: 0, 0, 0
minecraft:water
falling: false
#minecraft:water
```
- `Targeted Fluid`: 최대 거리 `20` 블록의 유체 좌표
- `id`: 유체의 이름
- `property`: 유체의 상태
- `tags`: 유체가 속한 `tags` 그룹

## memory
메모리 관련 정보를 표시합니다.
```js
Mem: 50% 1024/2048MB
Allocation rate: 000MB/s
Allocated: 50% 1024MB
```
- `Mem`: 점유율 현재사용량/전체할당량MB
- `Allocation rate`: 최근 `0.5`초 기준으로 `JVM`이 새로 할당한 메모리 속도
   - `GC`가 발생하면 순간적으로 줄어듦
- `Allocated`: 현재 `JVM`이 확보한 힙 메모리 총량

## particle_render_stats
입자의 정보를 표시합니다.
```js
P: 0
```
- `P`: 현재 보이는 입자의 개수
   - 최대 `16,384`개

## player_position
현재 플레이어의 좌표 관련 정보를 표시합니다.
```js
XYZ: 0.000 / 0.00000 / 0.000
Block: 0 0 0
Chunk: 0 0 0 [0 0 in r.0.0.mca]
Facing: south (Towards positive Z) (0.0 / 0.0)
minecraft:overworld FC: 0
```
- `XYZ`: 현재 x,y,z 좌표 소수점
- `Block`: 현재 위치의 블록 좌표
- `Chunk`: 현재 위치의 청크 좌표
   - `0 0 in r.0.0.mca`: 청크 파일 좌표
- `Facing`: 현재 바라보고 있는 방향
   - `0.0 / 0.0`: yaw, pitch
- `dimension`: 현재 속한 차원
   - `FC`: 속한 차원의 항상 로드되는 청크(`/forceload`로 인한)

## player_section_position
현재 플레이어의 섹션에서의 상대 좌표
```js
Section-relative: 00 00 00
```

## post_effect
```js
Post: 
```
알 수 없음

## simple_performance_impactors
화면 성능에 영향이 있는 설정을 표시합니다.
```js
improved-transparency fancy-clouds B: 0
Filtering: 이방성 8x
```
- `improved-transparency`: 비디오 설정-개선된 투명도(켜짐)
- `clouds`: 비디오 설정-구름
- `B`: 비디오 설정-생물 군계 혼합
- `Filtering`: 비디오 설정-텍스쳐 필터링

## sound_mood
소리와 관련한 정보를 표시합니다.
```js
Sounds: 0/247 + 1/8 (Mood 0%)
```
- `Sounds`: 현재 재생 중인 소리의 수/최대 재생 가능한 소리의 수
- `Mood`: 현재 재생 중인 분위기 음의 수/최대 재생 가능한 분위기 음의 수

## system_specs
현재 마인크래프트가 실행되는 컴퓨터 정보를 표시합니다.
```js
Java: 21.0.7
CPU: 6x Intel(R) Core(TM) i5-8500 CPU @ 3.00GHz
Display: 1920x1080 (Intel)
Intel(R) UHD Graphics 630
OpenGL 3.3.0 - Build 30.0.100.9805
```

## tps
현재 서버의 `mspt`와 관련된 정보를 표시합니다.
```js
Integrated server @ 0.0/50.0 ms, 20 tx, 1 rx
```
```js
MyServer server @ 0.0/50.0 ms, 20 tx, 1 rx
```
- `server`: 싱글 플레이라면 `Integrated server`
   - 멀티 플레이 서버라면 `<서버 이름> server`
- `mspt`: 현재 틱의 처리 시간/최대 틱 처리 시간
   - `(frozen)`: 현재 게임이 멈춘 경우(`/tick freeze`)
   - `(frozen - stepping)`: 현재 틱이 진행하는 경우(`/tick step <n>`)
   - `(sprinting)`: 현재 틱이 가속하는 경우
- `tx`: 초당 전송한 패킷의 수(과거 평균: `75%`, 현재 평균: `25%` 합산으로 계산)
- `rx`: 초당 수신한 패킷의 수(과거 평균: `75%`, 현재 평균: `25%` 합산으로 계산)
